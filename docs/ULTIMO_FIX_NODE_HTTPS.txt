CivicVois - Fix finale Node HTTPS

Questa versione corregge due problemi concreti visti durante il test:

1) La pagina #/auth caricava il feed segnalazioni anche prima di premere Accedi/Registrati.
   Se la lettura dati falliva, compariva subito il toast "fetch failed" anche se l'utente non aveva ancora fatto nulla.
   Ora in #/auth non vengono caricate segnalazioni finché non serve davvero.

2) La Netlify Function non usa più global fetch per parlare con Supabase.
   Usa il modulo Node https nativo. Questo evita errori generici "fetch failed" prodotti da Undici/fetch lato serverless.

Test corretti dopo il deploy:

- Health Function:
  https://keen-palmier-430d1a.netlify.app/.netlify/functions/civicvois-api?health=1

- Test Supabase dalla Function:
  https://keen-palmier-430d1a.netlify.app/.netlify/functions/civicvois-api?supabase=1

Se health=1 funziona ma supabase=1 no, il problema non è Netlify: è il collegamento Netlify Function -> Supabase URL/key/rete.
Se supabase=1 funziona, poi il login/registrazione deve passare attraverso la stessa Function.
