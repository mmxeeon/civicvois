CivicVois - versione definitiva con Netlify Function proxy

Questa build risolve il problema "Failed to fetch" spostando tutte le chiamate Supabase dietro una Netlify Function same-origin:
/.netlify/functions/civicvois-api

Perché:
- il browser non chiama più direttamente *.supabase.co;
- si evitano blocchi CORS/browser/estensioni/DNS locale sul dominio Supabase;
- login, registrazione, query database e upload storage passano da Netlify;
- nel frontend resta solo la publishable key, mai la secret key.

Dopo il deploy:
1. Carica lo ZIP su Netlify.
2. Apri prima https://keen-palmier-430d1a.netlify.app
3. Fai CMD+SHIFT+R.
4. Se Netlify mostra una sezione Functions con civicvois-api, la build è corretta.
5. Se vuoi testare da console: window.CV_DEBUG.dump() e await window.CV_DEBUG.testRest()

Non rieseguire schema.sql se è già stato eseguito.
La patch supabase/privileges-patch.sql l'hai già eseguita correttamente.
